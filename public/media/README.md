# Фото биографий

Все фотографии биографий лежат **здесь**. Next.js раздаёт эту папку
по адресу `/media/...` (потому что родитель — `public/`).

## Структура

Повторяем схему WordPress `wp-content/uploads/YYYY/MM/`:

```
public/media/
├── 2024/
│   └── 06/
│       ├── andrey-meyster-2.jpg
│       └── meyster-andrey-yurevich.jpg
├── 2025/
│   └── 04/
│       └── ...
└── ...
```

Почему с годом и месяцем:
- 1:1 совпадает с оригинальными путями на `lichnosty.ru` → при массовой миграции `scripts/import-uploads.js` просто копирует без ручных правок.
- Легко ориентироваться по дате загрузки.
- Не упираемся в лимиты количества файлов в одной папке на файловой системе.

## Правила именования

1. **Только латиница**, строчные буквы, через `-` (hyphen).
2. Желательно — **slug биографии** + опциональный суффикс:
   - `ivan-ivanov.jpg` — основное фото (идёт в `photo_url`).
   - `ivan-ivanov-2.jpg`, `ivan-ivanov-3.jpg` — для галереи.
   - `ivan-ivanov-avatar.jpg`, `ivan-ivanov-full.jpg` — по смыслу.
3. Кириллицу транслитерируем (`фёдор → fyodor`, `щукин → schukin`).
4. Пробелы и спецсимволы не используем.
5. Формат: `.jpg` / `.jpeg` / `.png` / `.webp`. WebP предпочтительнее
   для больших фото (меньше вес), но совместимость хуже — JPG безопаснее.

## Пропорции и размеры

- **Основное фото** (поле `photo_url`, рендерится в `.author_photo`
  и в карточках `.card-photo`): **3:4**, рекомендуется `600×800`
  или `900×1200`. Оригиналы `1024×1024` тоже подходят — CSS обрежет.
- **Галерея** (массив `photos`): любые пропорции, лучше одинаковые
  для ряда.

## Варианты размеров WordPress

WordPress автоматически генерирует `-150x150`, `-300x400`, `-scaled`,
`-thumbnail` копии. В наш репо **не надо** — держим только оригинал.
Next.js сам через `next/image` отдаст нужные размеры.

При массовой миграции запускайте `npm run import:uploads -- --strip-variants`
— скрипт выкинет все варианты и оставит оригиналы.

## Как использовать в биографии

В JSON-дампе (`migration/db/lico_export.json`) или при ручном добавлении
`Person` в БД указывайте **относительный путь**:

```json
{
  "thumbnail": "/media/2024/06/andrey-meyster-2.jpg",
  "attachments": [
    { "url": "/media/2024/06/meyster-andrey-yurevich.jpg", "mime": "image/jpeg" }
  ]
}
```

или в Prisma:

```js
await db.person.update({
  where: { slug: 'ivan-ivanov' },
  data: {
    photo_url: '/media/2025/04/ivan-ivanov.jpg',
    photos: ['/media/2025/04/ivan-ivanov-2.jpg']
  }
});
```

## Что НЕ коммитить

- Оригиналы > 2 МБ — сжимайте [squoosh.app](https://squoosh.app) или `cwebp`.
- `wp-content/uploads/` целиком — кидайте в `migration/uploads/`,
  оттуда скрипт перенесёт в `public/media/` с очисткой вариантов.
- `.DS_Store`, `Thumbs.db` — системный мусор.
